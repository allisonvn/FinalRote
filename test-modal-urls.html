🔧 Teste Modal URLs - RotaFinal

Instruções: Este teste verifica se as URLs estão aparecendo corretamente no modal "Detalhes do Experimento".

Passos para Testar:

  1. Abra o dashboard do RotaFinal
  2. Clique em "Detalhes" em qualquer experimento
  3. Vá para a aba "URLs e Configurações"
  4. Verifique se as URLs das variantes estão sendo exibidas
  5. Abra o console do navegador (F12) para ver os logs de debug

Logs de Debug Esperados:

🔍 Buscando dados das variantes para experimento: [experiment-id] 📊 Variantes encontradas: [array com variantes] 📋 Variante Controle processada: {id, name, redirect_url, ...} 📋 Variante Variante A processada: {id, name, redirect_url, ...} ✅ Dados das variantes processados: [array processado] 🔄 Definindo dados das variantes no estado: [array final] 🎨 Renderizando URLs com dados: [array de variantes] 🔗 URLs encontradas: [{name: "Controle", url: "..."}, {name: "Variante A", url: "..."}]

Possíveis Problemas e Soluções:

Problema 1: Modal não abre ou não carrega dados
Solução: Verifique se o experimento tem um project_id válido
Problema 2: Variantes não são encontradas
Solução: Verifique se as variantes foram criadas corretamente no banco
Problema 3: URLs são null ou vazias
Solução: As variantes podem não ter URLs configuradas ainda
✅ Correções Aplicadas:
  • Adicionados logs de debug para rastrear o carregamento das variantes
  • Corrigido o select na API para incluir redirect_url
  • Adicionada validação para experimentos sem project_id
  • Melhorada a exibição das URLs no modal

Como Verificar se Funcionou:

  1. Abra o console do navegador (F12)
  2. Navegue para o modal de detalhes do experimento
  3. Procure pelos logs que começam com 🔍, 📊, 📋, ✅, 🔄, 🎨, 🔗
  4. Verifique se as URLs estão sendo exibidas na aba "URLs e Configurações"
Nota: Se as URLs ainda não aparecerem, verifique se as variantes foram criadas com URLs válidas no banco de dados.
